TransParent Grid [ Demos ]

Introduction

The TransParent Grid is highly flexible, extremely powerful, fluid and responsive. It was brought about by a need for a more flexible grid system that was not bound to a set amount of columns (12 or 16, in the most common cases). Instead, it has a variable amount of columns per row ranging anywhere from 1 to 16. It also uses an easy-to-remember naming convention that allows for an extremely small learning-curve, has a few biased built-in extras, and is surprisingly flexible.

Quick-start Documentation

The keys to best-practice in effectively utilizing the grid:

  • Basic Usage
    • '.container': the outermost wrapper — it sets the overall max-width for all display sizes,
    • '.row-*': where '*' is equal to the number of columns this grid row will have,
    • '.col-*': where '*' is equal to the number of columns this column counts for of the grid row columns.
  • Separate the structure from the skin
    • don't add the grid classes to existing mark-up — add new mark-up to apply the grid,
    • don't add any other classes to the grid mark-up — unless you know exactly what you are doing, and…
    • don't overlap grid classes — don't add a 'row-*' class to mark-up that already has a 'col-*' class.

Compatibility: the TransParent Grid uses the border-box box-sizing method, so it is not compatible with IE7 and down. Otherwise, it should work in most other modern browsers (Chrome, FF, Opera, and Safari).

It should be able to nest any combination of the units and lines, and so it should be able to create the complicated layout below. His test is reminiscent of the golden mean, so it is still used today.1
